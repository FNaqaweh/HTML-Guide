Emphasizing Text

There are a two different ways to show emphasis in text in HTML. Functional emphasis, and aesthetic emphasis.

Functional Emphasis

Functional emphasis in HTML includes using the <strong> tag and the <em> tag.
The <strong> tag is used to strongly emphasize the text. This emphasis is functional because it allows the browser to understand that this piece of text is very significant, and not just bolded for design.
The <em> tag is used to emphasize the text. This emphasis is once again, functional, because it allows the browser to understand that the piece of text is significant, but not as significant as the text embedded within the <strong> tag

Aesthetic Emphasis

Aesthetic emphasis is different from functional emphasis because the main purpose is for aesthetic design, rather than functionality. Using the <b> tag and the <i> tag, developers may bolden or italicise the text without outlining it's significance to the browser.

Controlling Whitespace

By default, browsers will not take any whitespace past the first space into consideration.
For example, if i type 10 spaces between this words contained within single quotation 'marks,' 'the' white space will not be taken into account by the browser, as you can see on the screen.

Any extra desired whitespace can be typed in with a special character entity, which is the &nbsp; character entity. (to display the & symbol, use character entity &amp;)
What's special about the &nbsp; entity, is that it will also ensure that the browser will not seperate the words which surround the space through different lines. For example, try and seperate the two words 'Seperate Me!' by resizing the browser (unless I did not code the character entity correctly, you should not be able to do it!)

Structuring HTML Documents

Aside from just using headings, HTML5 now further supports website structure by including tags which will increase the quality of a web document's semantics.
These tags include the <nav> tag, the <article> tag, the <section> tag, and the <aside> tag.
Each of these tags enhance the structure of a web page by doing different things, however they add no aesthetic value.

The <nav> Tag

This tag wraps around the content of the page which is responsible for the document's navigation. This section includes ways to navigate to different pages or may navigate to different sections in the same page. It is a section just for navigation links.

The <article> Tag

The article tag wraps around all the main web document's content. It signifies the start and end of the content in each page.

The <section> Tag

The section tag wraps around each different section in a web document. This can also wrap around some headers which developers might have dedicated to indicate the start and end of each section.

The <aside> Tag

The aside tag wraps around a section which may not be directly related to the page content. This section is significant, however it is not the main focus of the page.

The <div>Tag

The div tag is used as a last resort! Developers use the <div> tag to group together content, rather than sectioning it. It adds no semantic meaning to the outline, unless used alongside a 'class' or 'id' attribute.
An example of a situation developers may use this tag for, is to group together content for stylistic effects.

Creating Links

Links are one of the most important aspects of HTML, and are the backbone of the internet. The following section will outline the syntax of creating links in HTML.

Syntax for Link Creation

In order to create a link, the <a> tag must surround the words the developer intents to make as a link.
The <a> tag stands for "anchor" tag, and must not be confused with the <link> tag, which is responsible for linking a reference for the HTML document, rather than creating a link in the webpage.
Links are surrounded by the <a> tag, however, several attributes must be included in the tag for it to navigate the internet.
Following the opening of the tag, the attributes will include the 'href' attribute (which will tell the browser where to find the HTML document you are linking to), and the 'title' attribute (which will give a brief description about the link).

This is how the final syntax should look like: <a href="(html file path OR full url)" title="(brief description)">content which will be the link</a>

Downloading a File From a Link

In order for a user to download a file from a webpage, the developer must reference the link to a file that the browser does not recognize (for example, a zip file). If the browser recognizes the file (for example a pdf), then the browser will open the file for browsing, rather than downloading. To indicate that a link is specifically for downloading a file, developers will add the 'download' attribute to the anchor tag. The value of the dowload attribute will be the name of the user's download.

For example, the syntax may look like <a href="cute_kitten.jpg" title="Very cute kitty!" download>insert link name here</a>

Linking to Different Sections Within a Webpage

Links also have the ability to redirect the user to different sections within the same webpage, or an external webpage. In order for a link to work like this, the 'href' attribute must point to a specific element within the HTML document. An 'id' attribute may be added to each section element, so that it may be easier for a link to point to it. For example, a developer me attach the 'id' attribute to the first heading and assign it the value of "first". An anchor tag can then link to that element, by assigning the value of the 'href' attribute to be a # followed by the id of the element the developer desires to link to. (e.g. 'href="#one"' if <h2 id="one"> was declared)
